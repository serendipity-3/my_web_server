# MESI 缓存一致性协议详解

> 多个 CPU 的缓存状态如何转换？总线事务如何保证一致性？

---

## 1. 基础概念

### 1.1 为什么需要缓存一致性？

```
┌─────────────────────────────────────────────────────────────┐
│  问题：每个 CPU 都有自己的缓存副本                           │
│                                                             │
│  CPU0 L1: counter = 0                                       │
│  CPU1 L1: counter = 0                                       │
│  内存:    counter = 0                                       │
│                                                             │
│  如果 CPU0 把 counter 改成 1：                              │
│  CPU0 L1: counter = 1  ← 已修改                             │
│  CPU1 L1: counter = 0  ← 过期数据！                         │
│  内存:    counter = 0  ← 也是旧值                           │
│                                                             │
│  缓存一致性协议保证：CPU1 能看到 CPU0 的修改                 │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 MESI 四种状态

```
┌─────────────┬────────────────────────────────────────────────────┐
│    状态      │                    含义                             │
├─────────────┼────────────────────────────────────────────────────┤
│ Modified    │ 这份数据只存在于我的缓存中，且我修改过它               │
│ (M)         │ 内存中的副本已经过时，我有责任写回                    │
│             │ 其他CPU的副本都是 Invalid                            │
├─────────────┼────────────────────────────────────────────────────┤
│ Exclusive   │ 这份数据只存在于我的缓存中，但我没修改过               │
│ (E)         │ 内存中的副本是最新的，我可以直接改成 Modified          │
│             │ 其他CPU的副本都是 Invalid                            │
├─────────────┼────────────────────────────────────────────────────┤
│ Shared      │ 这份数据可能存在于多个 CPU 的缓存中                   │
│ (S)         │ 所有副本和内存中的值一致                              │
│             │ 不能直接修改，需要先升级                              │
├─────────────┼────────────────────────────────────────────────────┤
│ Invalid     │ 这份缓存行无效，不能使用                              │
│ (I)         │ 其他 CPU 修改了数据，我的副本被作废                   │
└─────────────┴────────────────────────────────────────────────────┘
```

## 2. 总线事务

### 2.1 五种总线事务

```
┌─────────────┬────────────────────────────────────────────────────┐
│  总线事务    │                    含义                             │
├─────────────┼────────────────────────────────────────────────────┤
│ BusRd       │ 读取请求：我想读这个地址的数据                       │
│             │ 其他CPU如果有 Modified，需要写回                     │
├─────────────┼────────────────────────────────────────────────────┤
│ BusRdX      │ 独占读取请求：我想读且要独占修改                     │
│             │ 其他CPU的副本全部失效                                │
├─────────────┼────────────────────────────────────────────────────┤
│ BusUpgr     │ 升级请求：我有 Shared，想升级成 Modified             │
│             │ 其他CPU的副本失效，但不需要返回数据                  │
├─────────────┼────────────────────────────────────────────────────┤
│ BusFlush    │ 写回请求：我要把 Modified 写回内存                   │
│             │ 通常在缓存行被驱逐时使用                             │
├─────────────┼────────────────────────────────────────────────────┤
│ BusFlushX   │ 独占写回：写回并让其他CPU失效                       │
│             │ 用于缓存行被驱逐且需要独占时                         │
└─────────────┴────────────────────────────────────────────────────┘
```

### 2.2 总线事务触发的状态变化

```
┌───────────────┬───────────────┬───────────────┬───────────────┐
│  当前状态      │  自己读取      │  自己写入      │  其他CPU读取   │
├───────────────┼───────────────┼───────────────┼───────────────┤
│ Invalid (I)   │ BusRd → S/E  │ BusRdX → M   │ 无反应        │
├───────────────┼───────────────┼───────────────┼───────────────┤
│ Shared (S)    │ 直接读 → S   │ BusUpgr → M  │ 无反应        │
├───────────────┼───────────────┼───────────────┼───────────────┤
│ Exclusive (E) │ 直接读 → E   │ 直接写 → M   │ BusRd → S    │
├───────────────┼───────────────┼───────────────┼───────────────┤
│ Modified (M)  │ 直接读 → M   │ 直接写 → M   │ BusRd → S    │
│               │              │              │ (需要写回)     │
└───────────────┴───────────────┴───────────────┴───────────────┘
```

## 3. 完整状态转换图

```
                              CPU 读取
                                 │
                                 ▼
              ┌──────────────────────────────────┐
              │                                  │
         ┌────┴────┐    BusRd        ┌────┴────┐
    ┌───►│Invalid  │◄───────────────►│ Shared  │◄───┐
    │    │  (I)    │                 │  (S)    │    │
    │    └────┬────┘                 └────┬────┘    │
    │         │                           │         │
    │         │ BusRdX                    │ BusUpgr │
    │         │ (独占获取)                 │ (升级)  │
    │         │                           │         │
    │         ▼                           │         │
    │    ┌─────────┐                      │         │
    │    │Exclusive│                      │         │
    │    │  (E)    │                      │         │
    │    └────┬────┘                      │         │
    │         │                           │         │
    │         │ CPU 写入                  │ CPU 写入│
    │         │ (无需总线事务)             │(需要BusUpgr)
    │         │                           │         │
    │         ▼                           ▼         │
    │    ┌─────────────────────────────────────┐    │
    │    │           Modified (M)              │    │

    │    └──────────────────┬──────────────────┘    │
    │                       │                       │
    │                       │ 其他CPU读取           │
    │                       │ (BusRd，触发写回)     │
    │                       │                       │
    └───────────────────────┴───────────────────────┘
```

## 4. 详细场景分析

### 4.1 场景1：CPU0 读取，CPU1 读取

```
初始状态：
    所有缓存行都是 Invalid

时间 ──────────────────────────────────────────────────────────────►

CPU0                              CPU1                              缓存状态
────                              ────                              ────────

读取 x
  │
  ├─ 缓存未命中
  │
  ├─ 发送 BusRd
  │                               │                                 x: I,I
  │                               │
  │◄──────────────────────────────┤ 内存返回 x=0
  │
  ├─ 缓存行状态: Exclusive
  │                                 x: E(CPU0),I
  │                               │
  ▼                               │
                                  读取 x
                                    │
                                    ├─ 缓存未命中
                                    │
                                    ├─ 发送 BusRd
  │                               │
  │ 收到 BusRd                    │
  │ （我有 Exclusive，不需要写回）  │
  │ 状态变为 Shared               │
  │                               │
  │◄──────────────────────────────┤
  │                               │
  │                               ├─ 获得 x=0
  │                               │
  │                                 x: S,S
  ▼                               ▼
```

### 4.2 场景2：CPU0 读取，CPU0 写入

```
初始状态：
    所有缓存行都是 Invalid

CPU0                              缓存状态
────                              ────────

读取 x
  │
  ├─ 缓存未命中
  │
  ├─ 发送 BusRd
  │                                 x: I
  │
  ├─ 获得 x=0
  │
  ├─ 状态: Exclusive
  │                                 x: E
  │
  ▼
写入 x=42
  │
  ├─ 缓存命中（Exclusive）
  │
  ├─ 直接修改，不需要总线事务
  │
  ├─ 状态: Modified
  │                                 x: M
  │
  ▼
```

### 4.3 场景3：CPU0 修改，CPU1 读取（需要写回）

```
初始状态：
    CPU0: x=42, Modified
    CPU1: Invalid

CPU0                              CPU1                              缓存状态
────                              ────                              ────────

                                  读取 x
                                    │
                                    ├─ 缓存未命中
                                    │
                                    ├─ 发送 BusRd
  │                               │                                 x: M,I
  │                               │
  │ 收到 BusRd                    │
  │ （我有 Modified，必须写回！）   │
  │                               │
  ├─ 写回 x=42 到内存             │
  │                               │
  ├─ 状态变为 Shared              │
  │                               │
  │──────────────────────────────>┤ 返回 x=42
  │                               │
  │                                 x: S,S
  │                               │
  │                               ├─ 获得 x=42
  │                               │
  │                               ├─ 状态: Shared
  ▼                               ▼
```

### 4.4 场景4：Shared 时写入（需要升级）

```
初始状态：
    CPU0: x=0, Shared
    CPU1: x=0, Shared

CPU0                              CPU1                              缓存状态
────                              ────                              ────────

写入 x=42
  │
  ├─ 缓存命中（Shared）
  │
  ├─ 不能直接修改！
  │
  ├─ 发送 BusUpgr
  │                               │                                 x: S,S
  │                               │
  │                               ├─ 收到 BusUpgr
  │                               │
  │                               ├─ 状态变为 Invalid
  │                               │
  │◄──────────────────────────────┤
  │                               │                                 x: M,I
  │                               │
  ├─ 获得独占权
  │
  ├─ 状态变为 Modified
  │
  ├─ 修改 x=42
  │
  ▼                               ▼
```

## 5. 多CPU复杂场景

### 5.1 场景5：三个CPU同时读取

```
初始状态：
    所有缓存行都是 Invalid

CPU0                              CPU1                              CPU2                              缓存状态
────                              ────                              ────                              ────────

读取 x
  │
  ├─ BusRd
  │                               │                                 │                                 x: I,I,I
  │◄──────────────────────────────┤ 内存返回
  │                               │
  │ 状态: Exclusive               │                                 │                                 x: E,I,I
  │                               │                                 │
  ▼                               │                                 │
                                  读取 x
                                    │
                                    ├─ BusRd
  │                               │                                 │
  │ 收到 BusRd                    │                                 │
  │ 状态: Shared                  │                                 │
  │                               │                                 │                                 x: S,S,I
  │◄──────────────────────────────┤ 返回 x
  │                               │                                 │
  │                               │ 状态: Shared                   │
  │                               │                                 │
  │                               ▼                                 │
                                                                  读取 x
                                                                    │
                                                                    ├─ BusRd
  │                               │                                 │
  │ 收到 BusRd（已经是Shared）     │                                 │
  │ 保持 Shared                   │                                 │
  │                               │                                 │                                 x: S,S,S
  │◄──────────────────────────────┼─────────────────────────────────┤
  │                               │                                 │
  │                               │                                 ├─ 状态: Shared
  │                               │                                 │
  ▼                               ▼                                 ▼
```

### 5.2 场景6：三个CPU，一个写入

```
初始状态：
    CPU0: x=0, Shared
    CPU1: x=0, Shared
    CPU2: x=0, Shared

CPU0                              CPU1                              CPU2                              缓存状态
────                              ────                              ────                              ────────

写入 x=42
  │
  ├─ BusUpgr
  │                               │                                 │                                 x: S,S,S
  │                               │                                 │
  │                               ├─ BusUpgr 收到                   │                                 
  │                               │ 状态: Invalid                  │
  │                               │                                 │
  │                               │                                 ├─ BusUpgr 收到
  │                               │                                 │ 状态: Invalid
  │                               │                                 │
  │◄──────────────────────────────┼─────────────────────────────────┤ 广播完成
  │                               │                                 │
  │ 状态: Modified                │                                 │                                 x: M,I,I
  │                               │                                 │
  ▼                               ▼                                 ▼
```

### 5.3 场景7：CPU0 修改中，CPU1 和 CPU2 同时读取

```
初始状态：
    CPU0: x=42, Modified
    CPU1: Invalid
    CPU2: Invalid

CPU0                              CPU1                              CPU2                              缓存状态
────                              ────                              ────                              ────────

                                  读取 x                            读取 x
                                    │                                 │
                                    ├─ BusRd                         ├─ BusRd
  │                               │                                 │                                 x: M,I,I
  │                               │                                 │
  │ 两个 BusRd 同时到达            │                                 │
  │ 总线仲裁器决定顺序             │                                 │
  │                               │                                 │
  │ 收到 BusRd（CPU1先）           │                                 │
  │                               │                                 │
  ├─ 写回 x=42 到内存             │                                 │
  │                               │                                 │
  ├─ 状态: Shared                 │                                 │
  │                               │                                 │
  │◄──────────────────────────────┤ 返回 x=42                       │                                 x: S,S,I
  │                               │                                 │
  │                               │ 状态: Shared                   │
  │                               │                                 │
  │                               │                                 │
  │                               │                                 │ BusRd（CPU2）
  │                               │                                 │
  │ 收到 BusRd（已经是Shared）     │                                 │
  │ 保持 Shared                   │                                 │
  │                               │                                 │                                 x: S,S,S
  │◄──────────────────────────────┼─────────────────────────────────┤
  │                               │                                 │
  │                               │                                 │ 状态: Shared
  │                               │                                 │
  ▼                               ▼                                 ▼
```

## 6. 状态转换的完整矩阵

```
┌────────────┬──────────────────┬──────────────────┬──────────────────┬──────────────────┐
│ 当前状态    │ 自己读取          │ 自己写入          │ 其他读取          │ 其他写入          │
├────────────┼──────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ Invalid    │ 发BusRd          │ 发BusRdX         │ 无               │ 无               │
│ (I)        │ → S或E           │ → M              │                  │                  │
├────────────┼──────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ Shared     │ 直接读           │ 发BusUpgr        │ 无               │ 发BusUpgr        │
│ (S)        │ → S              │ → M              │                  │ → I              │
├────────────┼──────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ Exclusive  │ 直接读           │ 直接写           │ 发BusRd          │ 发BusRdX         │
│ (E)        │ → E              │ → M              │ → S              │ → I              │
├────────────┼──────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ Modified   │ 直接读           │ 直接写           │ 收BusRd，写回    │ 收BusRdX，写回   │
│ (M)        │ → M              │ → M              │ → S              │ → I              │
└────────────┴──────────────────┴──────────────────┴──────────────────┴──────────────────┘
```

## 7. 缓存行乒乓效应

```
┌─────────────────────────────────────────────────────────────┐
│  当多个CPU频繁修改同一变量时：                               │
│                                                             │
│  CPU0 修改 → M                                              │
│  CPU1 要修改 → BusUpgr → CPU0写回 → S → BusUpgr → M        │
│  CPU2 要修改 → BusUpgr → CPU1写回 → S → BusUpgr → M        │
│  CPU0 又要修改 → BusUpgr → CPU2写回 → S → BusUpgr → M      │
│                                                             │
│  每次修改都要：                                              │
│  1. 使其他CPU的缓存失效                                     │
│  2. 等待写回完成                                            │
│  3. 获取独占权                                              │
│                                                             │
│  这就是为什么高竞争的原子操作很慢                            │
└─────────────────────────────────────────────────────────────┘
```

## 9. 总结

```
┌─────────────────────────────────────────────────────────────┐
│  MESI 协议的核心：                                           │
│                                                             │
│  1. 四种状态：M, E, S, I                                    │
│  2. 五种总线事务：BusRd, BusRdX, BusUpgr, BusFlush, BusFlushX│
│  3. 状态转换由总线事务触发                                    │
│  4. Modified 状态的写回保证一致性                            │
│  5. 总线仲裁器保证顺序                                       │
│                                                             │
│  不需要软件干预，硬件自动保证缓存一致性                      │
└─────────────────────────────────────────────────────────────┘
```
